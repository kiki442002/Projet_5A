STM32CubeWB Firmware Package V1.9.0 / 25-Septembre-2020



Main Changes

           Introduce BLE/Thread and BLE/Zigbee dynamic concurrent mode, the support of LLD BLE and PHY_802.15.4 CLI application

  • Dynamic Concurrent mode support between BLE and Zigbee
    • Through this feature, the STM32WB can support Zigbee and BLE protocols in parallel.
    • 3 new applications available on P-NUCLEO-WB55.Nucleo board highlight this feature.
    • They are available under Projects-NUCLEO-WB55.Nucleo_Zigbee
      • BLE_Zigbee_Dyn
      • BLE_Zigbee_Dyn_NVM
      • BLE_Zigbee_Dyn_SED
    • These application are using the following wireless copro binaries:
      • stm32wb5x_BLE_Zigbee_FFD_dynamic_fw.bin : BLE Zigbee Full Feature Device Dynamic concurrent mode. To be used for Zigbee Coordinator or Router devices sharing its radio bandwidth with BLE.
      • stm32wb5x_BLE_Zigbee_RFD_dynamic_fw.bin : BLE Zigbee Reduced Feature Device Dynamic concurrent mode. To be used for Zigbee End Devices sharing its radio bandwidth with BLE.
  • Dynamic Concurrent mode support between BLE and Thread
    • Through this feature, the STM32WB can support Thread and BLE protocols in parallel.
    • 2 new applications available under Projects-NUCLEO-WB55.Nucleo_Thread highlight this feature.
      • Ble_Thread_Dyn
      • Ble_Thread_Dyn_SED
    • These application are using the following wireless copro binary: stm32wb5x_BLE_Thread_FFD_dynamic_fw.bin.
  • Static concurrent mode support between BLE and Zigbee-RFD
    • When using the static concurrent mode, the application can switch from BLE to Zigbee or from Zigbee to BLE on request.
    • There are two flavors of static concurrent mode supporting BLE and Zigbee:
      • BLE-Zigbee_FFD (Full Function Device)
      • BLE-Zigbee_RFD (Reduced Function Device and optimized for power consumption)
    • The stm32wb5x_BLE_Zigbee_RFD_static_fw.bin copro binary is to be used for Zigbee End devices requesting BLE support in static concurrent mode.
  • Provide the ability to do only BLE advertising & scanning at HCI level
  • PHY_802.15.4 command line interface application
    • A dedicated application allows the control and the test of the 802_15_4 radio via a command line interface.
    • It is available under Projects-NUCLEO-WB55.Nucleo_802_15_4
      • Phy_802_15_4_Cli
    • This application request the usage of the following wireless copro binary: stm32wb5x_Phy_802_15_4_fw.bin.
  • BLE and LLD
    • 3 new applications are provided based on usage of stm32wb5x_BLE_LLD_fw.bin:
      • LLD_BLE_Chat: Text communication between 2 boards using Terminal based on UART
      • LLD_BLE_Pressbutton: Transmission and Reception between 2 boards using button and symbolized by blinking LED
      • LLD_BLE_Proximity: State Machine Programmed to receive ID from others board (multi boards)
  • Thread
    • ID 91267 : Fix CoAP OpenThread API exposed on application side
      • The CoAP API is now compliant with OpenThread definition and mContext parameter is then correctly returned to the application.
