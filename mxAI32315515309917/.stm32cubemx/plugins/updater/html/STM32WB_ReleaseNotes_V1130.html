STM32CubeWB Firmware Package V1.13.0 / 12-November-2021
Main Changes

  • BLE updates:
    • Multi link support
    • L2CAP connection-oriented channels support (IP over BLE enabler)
    • ACI_GAP_START_NAME_DISCOVERY_PROC command is deprecated
  • 802.15.4 general updates (Zigbee, Thread, MAC, PHY)
    • CCA (Clear Channel Assessment) Threshold has been increased
  • Zigbee updates
    • 802_15_4 timer improvement solving several issues relative to the wraparound of this timer
  • Thread updates
    • Fix wrapper issue with UDP receivers
    • RCP build: improved commissioning process robustness
  • MAC 802.15.4 updates
    • Fix unicast data processing after restart from persistence
    • Fix wrong erratic PANID Compression Flag on Beacon Request
    • Reduce MaxBe low level threshold allowing user to change CSMA-CA values
  • BLE-Zigbee Concurrent mode
    • BLE-Zigbee static concurrent mode not supported in this release, will be fixed in the next release. Alternative is to use BLE-Zigbee Dynamic concurrent mode
  • Safeboot
    • New binaries added: stm32wb5x_Safeboot_fw.bin, stm32wb3x_Safeboot_fw.bin, stm32wb1x_Safeboot_fw.bin.
    • Safeboot is an independent part of the FUS that manages specifically one case: option bytes corruption.
    • When option bytes are corrupted, the STM32WB hardware forces the boot to Safeboot whatever the running firmware.
    • Safeboot is already present in the FUS and this image allows to update it. Refer to AN5185 for more details.
    • Safeboot upgrade is optional.
